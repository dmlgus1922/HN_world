광인사 최종프로젝트에 쓸 지도 API 연습 중.

어디에 쓰느냐:
1. 사용자가 일정을 등록할 때 출발장소, 도착장소를 받을 것임
2. 각각의 장소는 카카오 맵 api에서 검색
3. 마커가 생기는 위치를 사용자가 클릭했을 때 그곳의 도로명 주소 DB에 저장

4. 출발과 도착 장소 사이에 걸리는 시간은
    -> https://map.kakao.com/?sName='출발 도로명 주소'&eName='도착 도로명 주소' redirect
    -> 그 안에서 걸리는 시간 찾기
    -> 사용자가 자차 끌고 가면 car 클래스 클릭, 대중교통 타면 transit 클래스 클릭
        -> 필요 없을듯?
    -> 그 안에서 걸리는 시간 빼오기
    자동차: document.querySelector('.CarRouteSummaryView .time').textContent
    버스: document.querySelector('.TransitRouteItem .time').textContent

5. 가져온 시간 DB에 저장 
    -> 근데 교통 정보가 실시간으로 달라져서 이걸 저장하는게 맞는건가 싶음..
    -> 데일리 루틴이면 매번 같은 시간에 이동하는 거니까 처음 저장할 때 그 시간데에 시간 정보를 가져오는 것이라면 괜찮을지도

6. 사용자가 일정 등록할 때 시작 시각을 등록할 것임.
    -> 시작 시각 - 이동 시간 - 사용자가 설정(혹은 우리가 임의로 설정)한 시간에 알림
